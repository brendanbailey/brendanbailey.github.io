2016 Iowa Liquor Sales Forecast¶

By Brendan Bailey 3/28/17¶

Executive Summary:¶

The Iowa State Liquor Board gave us a dataset of 2015 liquor sales in order to predict 2016 liquor sales. The data shows if there is a \$1 increase in Q1 Sales, total sales will increase by about \$4. Applying this model to 2016 Q1 sales, I estimate that Total Sales will be \$274 million plus or minus \$11.8 million dollars (between \$263 million and \$286 million). This is compared to \$283 million in sales for 2015. An increase in total sales is possible, but total sales for 2016 are more likely to decrease.

I recommend that the Iowa State Liquor Board refrain from raising liquor taxes, if the decision was predicated on an increase in total sales.

Below are the steps I took to come to this conclusion:

def clean_iowa_data(df): #Cleans Dataframe - fills nulls, eliminates dupes, converts data types
    #Replacing spaces with underscores and deleting parentheses in column names
    df.columns = [column.replace(" ", "_").replace("(","").replace(")","") for column in df.columns]
    
    #Elimininating Duplicate Data
    df.drop_duplicates(inplace = True)
    
    #Third try at imputing empty data - 428 Nulls for County in 10% dataset!
    temp_df = iowa_data[["Store_Number", "County"]].dropna().drop_duplicates()
    temp_df.rename(columns = {"County": "Full_County"}, inplace = True)
    assert max(temp_df.groupby('Store_Number').Full_County.count()) == 1
    df = pd.merge(df, temp_df, how = "left", left_on = "Store_Number", right_on = "Store_Number")
    
    #Third try at imputing empty data - 0 improvement for Category Name on 10% dataset. Maybe increase improvement on full?
    temp_df = iowa_data[["Category", "Category_Name"]].dropna().drop_duplicates()
    temp_df.rename(columns = {"Category_Name": "Full_Category_Name"}, inplace = True)
    assert max(temp_df.groupby('Category').Full_Category_Name.count()) == 1
    df = pd.merge(df, temp_df, how = "left", left_on = "Category", right_on = "Category")
    
    #Converting column types
    df["Date"] = pd.to_datetime(df.Date)
    df["Sale_Dollars"] = df["Sale_Dollars"].replace( '[\$,)]','', regex=True ).astype(float)
    df["State_Bottle_Retail"] = df["State_Bottle_Retail"].replace( '[\$,)]','', regex=True ).astype(float) #http://stackoverflow.com/questions/31521526/pandas-column-convert-currency-to-float
    df["State_Bottle_Cost"] = df["State_Bottle_Cost"].replace( '[\$,)]','', regex=True ).astype(float)
    return df

To make the predictions of sales by store, we need to aggregate the data at the store level. Since we will be making predictions on Q1 data, variables specifically regarding Q1 were created.

There were some stores that did not have any 2015 Q1 sales in the 10% dataset, so I imputed the county average. However, that didn't seem to be a problem in the full dataset.

Data is also separated by year because we will be training and testing the model on the 2015 data and making predictions on the 2016 data.

Deduplicating by store number is an additional step taken to clean the data, as there are duplicates with slightly different geographic information.

def transform_data(df, year): #Transforms the DataFrame so it has aggregates at the Store Level
    base_df = df[df.Date.dt.year == year] #Selects the appropriate year
    master_df = base_df[["Store_Number", "City", "Zip_Code", "Full_County"]].drop_duplicates() #Adding Geographic Information
    
    group_by_list = ["Sale_Dollars", "Volume_Sold_Liters", "Bottles_Sold"] #Adding Summable Info
    for column in group_by_list:
        temp_df = base_df.groupby('Store_Number')[column].sum().to_frame()
        master_df = pd.merge(master_df, temp_df, how = "left", left_on = "Store_Number", right_index = True)
    
    group_by_list = ["State_Bottle_Cost", "State_Bottle_Retail"] #Adding Average Info
    for column in group_by_list:
        temp_df = base_df.groupby('Store_Number')[column].mean().to_frame()
        master_df = pd.merge(master_df, temp_df, how = "left", left_on = "Store_Number", right_index = True)
        
    group_by_list = ["Sale_Dollars", "Volume_Sold_Liters", "Bottles_Sold"] #Adding January through March Sales Data
    for column in group_by_list:
        temp_df = base_df[base_df.Date.dt.month.isin([1,2,3])] 
        temp_df = temp_df.groupby('Store_Number')[column].sum()
        temp_df.name = "Jan_Mar_%s" % column
        temp_df = temp_df.to_frame()
        master_df = pd.merge(master_df, temp_df, how = "left", left_on = "Store_Number", right_index = True)
        
        #Filling in Nulls for Jan Mar Data with County Average
        temp_df = master_df.groupby("Full_County")["Jan_Mar_%s" % column].mean()
        temp_df.name = "Jan_Mar_Cty_Avg_%s" % column
        temp_df = temp_df.to_frame()
        master_df = pd.merge(master_df, temp_df, how = "left", left_on = "Full_County", right_index = True)
        master_df["Jan_Mar_%s" % column].fillna(master_df["Jan_Mar_Cty_Avg_%s" % column], inplace = True)

    #Add Interaction for Jan Mar Sales Dollars and Jan Mar Volume Sold Liters
    master_df["Sales_Liters_Interaction"] = master_df["Sale_Dollars"] * master_df["Volume_Sold_Liters"]
    
    #Get County Dummies
    master_df.reset_index(inplace = True)
    temp_df = pd.get_dummies(master_df.Full_County)
    master_df = pd.merge(master_df, temp_df, how = "left", left_index = True, right_index = True)
    
    master_df.drop_duplicates(subset = "Store_Number", inplace = True) #Some Store Numbers are duplicated due to minor differences in location data
    return master_df

My initial set of variables were Q1 Sales, Q1 Volume Sold, Q1 Bottles Sold, and whether they were in Polk, Linn, or Black Hawk County (the top 3 counties for liquor most liquor transactions according to the 10% dataset). The county dummy variables were proven insignficant so they were thrown out.

Additionally, Q1 Volume Sold and Q1 Bottles Sold had a high variable inflation factor, so I threw those out too because I didn't want a multicollinearity problem.

I kept Q1 Sales and didn't seek any additional variables because with just Q1 Sales, I achieved a 0.97 R-Squared. I think this keeps in line with Ockham's Razor where we should pick a simple model over a complicated one.
